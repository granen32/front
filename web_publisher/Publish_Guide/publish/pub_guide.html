메타 태그의 종류와 규칙

프로젝트 환경 정의
<meta charset="utf-8"> 문자셋을 의미하며 공통적인 utf-8을 사용할 예정
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" /> 클라이언트와 서버간의 통신에서 주고 받는 http-equiv 속성으로 콘텐츠 타입을 지정해줌
<meta http-equiv="X-UA-Compatible" content="IE=edge;Chromeframe=1"> IE8이 개발되면서 생긴 호환성 모드에 대응하기 위한 속성이다. IE=EDGE는 항상 최신의 표준엔진을 사용해 렌더링 하게 된다.
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no" /> 화면 사이즈와 맞춰진다 모바일 브라우저에서 ZOOM을 사용하게 되는 데 최적의 사용환경을 위해 이를 방지해야 할경우 속성을 1로하고 사용자의 확대여부를 막는다.
<meta name="keywords" content=""> keyword 항목은 사이트내의 간략한 키워드를 뜻함
<meta name="description" content=""> description 항목은 사이트내의 간략한 설명을 뜻함
<meta name="robots" content="ALL"> 로봇 메타태그를 사용하여 구글 검색결과에서 사용자에게 게재되는 방식을 제어하는 자세한 페이지별 접근방식을 활용할 수 있음 기존은 all을 사용함
<meta name="author" content=""> author 항목은 문서의 저자를 정의하는 예제
<meta name="generator" content=""> 문서 작성에 사용한 저작 도구를 지정한다. 저작자가 저작 도구를 사용하지 않고 직접 작성한 때에는 이 속성값을 사용하지 않는다.
<meta http-equiv="refresh" content=""> 문서의 새로고침을 지정한다. 이값은 초단위의 시간주기를 의미하고 이때 url을 지정하면 새로고침과 동시에 url에 있는 문서로 자동 이동한다.
<meta property="og:type" content="">
<meta property="og:title" content="">
<meta property="og:description" content="">
<meta property="og:image" content="">
<meta property="og:url" content="">		 OPEN GRAPH 메타태그의 경우 페이스북에 표출되어지는 메타태그이다.
<meta name="twitter:title" content="" />
<meta name="twitter:url" content="" />
<meta name="twitter:image" content="" />
<meta name="twitter:description" content="" />		 트위터에서 표출되어지는 메타태그를 뜻한다.
<link rel="canonical" href="미정"> 중복 url 방지를 위한 태그
